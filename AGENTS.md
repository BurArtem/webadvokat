# Инструкции для AI-ассистента

## Git и сохранение изменений

**ВАЖНО: После каждого значимого изменения делай коммит!**

После внесения изменений в файлы, ВСЕГДА выполняй:

```bash
git add -A && git commit -m "Описание что было сделано"
```

### Примеры коммитов:
- `git add -A && git commit -m "Добавлено фото адвоката в hero-секцию"`
- `git add -A && git commit -m "Обновлены стили для hero-image"`
- `git add -A && git commit -m "Исправлена адаптивность на мобильных"`

### Как откатить изменения:

**Откатить всё к последнему коммиту:**
```bash
git checkout .
```

**Посмотреть историю коммитов:**
```bash
git log --oneline
```

**Откатиться на N коммитов назад:**
```bash
git reset --hard HEAD~1  # на 1 коммит назад
git reset --hard HEAD~2  # на 2 коммита назад
```

**Откатиться к конкретному коммиту:**
```bash
git checkout <hash-коммита>  # временно переключиться
git reset --hard <hash-коммита>  # permanently откатиться
```

## Правила работы с кодом

1. Перед редактированием файлов - читай их структуру
2. После изменений - проверяй результат в браузере
3. Делай коммиты после каждого логического изменения
4. Не коммить секреты и ключи API
